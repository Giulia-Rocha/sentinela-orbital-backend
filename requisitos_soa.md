GLOBAL SOLUTION 2026.1 – (SOA)
Tema: Space Connect – Tecnologia Espacial Aplicada a Desafios Reais
Contexto
A exploração espacial deixou de ser apenas um tema científico e passou a desempenhar um papel estratégico na sociedade moderna. Tecnologias espaciais, dados de satélites e sistemas de monitoramento global são utilizados para resolver problemas relacionados à agricultura, logística, sustentabilidade, clima, saúde, conectividade e gestão de riscos.

Diante desse cenário, sua equipe deverá propor e desenvolver uma solução tecnológica inovadora que conecte o ecossistema espacial a desafios reais enfrentados pela sociedade, utilizando os conceitos estudados ao longo do semestre na disciplina de (SOA).

O foco desta Global Solution será a construção de uma arquitetura orientada a serviços, promovendo interoperabilidade, integração de sistemas e comunicação entre aplicações por meio de APIs REST e Web Services SOAP.

Desafio
Desenvolver uma solução baseada em um dos temas relacionados à economia espacial, capaz de gerar impacto positivo na sociedade por meio do uso de tecnologias, dados ou serviços oriundos do setor espacial.

A solução deverá demonstrar como diferentes sistemas podem se comunicar e compartilhar informações utilizando serviços distribuídos.

A aplicação poderá ser voltada para a Terra, para o espaço ou para a integração entre ambos os ambientes.

Exemplos de Temas
Monitoramento agrícola com dados de satélite;

Previsão climática e prevenção de desastres naturais;

Monitoramento de queimadas e desmatamento;

Logística inteligente baseada em geolocalização;

Telemedicina para regiões remotas;

Rastreamento de recursos naturais;

Sistemas de apoio à agricultura sustentável;

Conectividade via satélite;

Simulação de colonização da Lua ou Marte;

Plataformas de análise de dados espaciais;

Soluções para cidades inteligentes.

O grupo poderá propor um tema próprio, desde que esteja alinhado à temática Space Connect.

Requisitos Técnicos Obrigatórios
1. Arquitetura Orientada a Serviços (SOA)
A solução deverá demonstrar claramente os princípios da Arquitetura Orientada a Serviços:

Baixo acoplamento;

Reutilização de serviços;

Interoperabilidade;

Contratos de serviço;

Integração entre sistemas;

Separação de responsabilidades.

2. API REST
Desenvolver uma API REST utilizando Java e Spring Boot contendo:

CRUD completo de uma entidade principal do domínio;

Utilização dos métodos HTTP adequados:

GET

POST

PUT

DELETE

Respostas em formato JSON;

Tratamento de erros e validações.

3. Web Service SOAP
Desenvolver um Web Service SOAP contendo ao menos:

Uma operação de consulta;

Uma operação de cadastro ou processamento;

Disponibilização do WSDL;

Testes de consumo via SoapUI ou ferramenta equivalente.

4. Integração entre Serviços
A solução deverá obrigatoriamente demonstrar:

Comunicação entre a API REST e o Web Service SOAP;
ou

Consumo de um serviço externo REST ou SOAP integrado ao projeto.

Exemplos:

APIs da NASA;

APIs do CPTEC/INPE;

Serviços climáticos;

Serviços de geolocalização;

Serviços de monitoramento ambiental.

5. Persistência de Dados
Utilizar:

Banco de dados H2 ou qualquer outro BD;

Operações CRUD completas;

Persistência dos dados da aplicação.

6. Programação Orientada a Objetos
Aplicar obrigatoriamente:

Abstração;

Encapsulamento;

Herança;

Polimorfismo.

7. Documentação dos Serviços
A equipe deverá documentar:

API REST
Endpoints disponíveis;

Métodos HTTP;

Exemplos de requisição;

Exemplos de resposta.

SOAP
Operações disponíveis;

Estruturas XML utilizadas;

WSDL do serviço;

Exemplos de requisição e resposta SOAP.

ODS Recomendados
A solução deverá estar relacionada a pelo menos um dos seguintes Objetivos de Desenvolvimento Sustentável:

ODS 1 – Fome Zero e Agricultura Sustentável;

ODS 2 – Trabalho Decente e Crescimento Econômico;

ODS 3 – Indústria, Inovação e Infraestrutura;

ODS 4 – Cidades e Comunidades Sustentáveis;

ODS 5 – Ação Contra a Mudança Global do Clima.

Diferenciais
Serão considerados diferenciais:

Integração com múltiplos serviços externos;

Gateway de APIs;

Mensageria (RabbitMQ, Kafka ou equivalente);

Docker;

Deploy em nuvem (AWS, Azure ou GCP);

Dashboard Web;

Testes automatizados;

Monitoramento dos serviços;

Aplicação de Inteligência Artificial;

Funcionalidades além das solicitadas.
